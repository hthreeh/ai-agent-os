# 操作系统智能代理 - 演示指南

## 演示前准备

### 1. 环境配置

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 API Key
cp .env.example .env
# 编辑 .env 文件，填入你的 OpenAI API Key
# OPENAI_API_KEY=sk-your-api-key-here

# 3. 启动服务
python src/main.py --mode web
```

### 2. 演示环境检查

- 确保 Linux 服务器可以正常执行命令
- 确认 OpenAI API Key 有效
- 测试基础命令：`df -h`, `ps aux`, `uname -a`
- 准备演示用的虚拟机或测试环境

## 演示流程（推荐顺序）

### 第一幕：基础功能展示（2-3分钟）

**目标**：展示系统信息查询和基本命令执行

#### 1.1 环境感知演示
```
输入：查看系统信息
```
**展示点**：
- 系统自动检测 OS 类型（Ubuntu/CentOS/openEuler）
- 右侧面板显示环境信息（OS 类型、主机名、内核版本）
- 缓存机制（TTL 3600秒，避免重复检测）

#### 1.2 系统资源查询
```
输入：查询磁盘使用情况
```
**展示点**：
- AI 思考动画（三个跳动圆点）
- 打字机效果输出结果
- 清晰格式化显示

```
输入：查看当前运行的进程
输入：查看内存使用情况
输入：查看 CPU 使用情况
```

#### 1.3 文件和端口查询
```
输入：搜索 /etc 目录下的所有 .conf 文件
输入：查看开放的端口
```
**展示点**：
- 槽位记忆：下次搜索自动继承目录
- 多种搜索模式支持

---

### 第二幕：连续任务编排（3-4分钟）

**目标**：展示任务分解、依赖管理和执行流程

#### 2.1 多步骤任务
```
输入：先查看磁盘使用情况，然后查看进程状态
```
**展示点**：
- 自动识别中文分隔符（先...然后...）
- 任务序列可视化（步骤进度条）
- 每个任务的状态追踪（pending/in_progress/completed）
- 最终综合结果汇总

#### 2.2 复杂任务链
```
输入：查看磁盘使用情况，并且查看进程状态，然后查看端口
```
**展示点**：
- 支持多种分隔符（并且、接着、最后）
- 任务执行顺序正确
- 每个任务独立结果展示

---

### 第三幕：条件分支和智能决策（3-4分钟）

**目标**：展示条件判断、LLM 任务分解和智能路由

#### 3.1 条件分支场景（场景一）
```
输入：检查磁盘空间，如果不足就清理日志，然后安装 nginx
```
**展示点**：
- LLM 智能分解复杂任务
- 条件分支逻辑（如果...就...）
- 依赖关系：清理后才能确保安装空间
- 任务执行顺序可视化

**预期任务链**：
1. 检查磁盘空间 (`df -h`)
2. 条件判断：磁盘使用率是否 > 80%
3. 如果不足：清理日志 (`find /var/log -name "*.log" -mtime +30 -delete`)
4. 安装 nginx (`apt-get install -y nginx`)

#### 3.2 环境感知和预检
```
输入：先检查磁盘空间，如果空间不足就清理日志文件
```
**展示点**：
- 环境预检机制（执行前检查条件）
- 条件分支执行（满足/不满足走不同路径）
- 预检失败时的跳过/中止策略

---

### 第四幕：用户管理和安全控制（3-4分钟）

**目标**：展示用户操作、权限管理和风险控制

#### 4.1 用户创建和权限配置（场景二）
```
输入：创建新用户 dev1，配置 sudo 权限，部署工作目录
```
**展示点**：
- 顺序依赖：用户必须先创建才能配置权限
- 参数提取：从自然语言中提取用户名
- 工作目录自动部署

**预期任务链**：
1. 创建用户 (`useradd dev1`)
2. 配置 sudo 权限 (`usermod -aG sudo dev1`) - 中风险需确认
3. 部署工作目录 (`mkdir -p /home/dev1/workspace && chown dev1:dev1`)

#### 4.2 风险控制演示
```
输入：删除用户 testuser
```
**展示点**：
- 中风险操作弹出确认对话框
- 风险解释和潜在影响说明
- 用户可以确认或取消

```
输入：rm -rf /tmp/test
```
**展示点**：
- 高风险命令自动拦截
- 安全事件记录到审计日志
- 详细的阻断原因说明

---

### 第五幕：故障诊断和排查（2-3分钟）

**目标**：展示多步诊断和推理能力

#### 5.1 端口排查（场景三）
```
输入：排查80端口无法访问的原因
```
**展示点**：
- 多步诊断流程
- 服务状态检查 (`systemctl status nginx`)
- 端口监听检查 (`ss -tuln | grep :80`)
- 防火墙规则检查 (`ufw status`)
- 综合分析结果

**预期任务链**：
1. 检查端口监听状态
2. 检查 Web 服务状态（nginx/apache2）
3. 检查防火墙规则
4. LLM 综合分析并给出诊断建议

---

### 第六幕：高级功能和用户体验（2-3分钟）

**目标**：展示系统的高级特性和优秀 UX

#### 6.1 会话管理
**展示点**：
- 左侧会话列表
- 新建对话（自定义名称）
- 切换不同会话
- 删除不需要的会话
- 会话统计（消息数、会话数）

#### 6.2 环境信息面板
**展示点**：
- 实时显示的操作系统类型
- 主机名和内核版本
- 会话统计信息
- 响应式设计（移动端自动隐藏）

#### 6.3 错误处理和重试
**展示点**：
- 命令执行失败自动重试（最多3次）
- 超时保护（30秒）
- 友好的错误提示

#### 6.4 审计日志

- 完整的操作轨迹记录
- 安全事件审计

## 演示技巧

### 1. 演示节奏控制
- 每个场景 2-3 分钟
- 重点展示评分相关的功能
- 避免长时间等待（命令执行快时适当解释）

### 2. 评分点对应
| 评分维度 | 演示对应 |
|---------|---------|
| 功能实现 (50%) | 所有基础命令执行 |
| 风险控制 (15%) | 风险确认和高危拦截 |
| 连续任务 (15%) | 场景一二三 |
| 环境感知 (5%) | 右侧环境面板 |
| 状态更新 (10%) | 任务序列可视化 |
| 性能 (10%) | 快速响应和缓存 |
| 稳定性 (10%) | 错误处理和重试 |
| 用户体验 (30%) | UI/UX 展示 |
| 工程质量 (30%) | 代码结构展示 |
| 创新性 (30%) | LLM 分解和条件分支 |

### 3. 备用演示方案
如果某些功能在演示环境受限：
- **用户管理**：可以用 `echo` 命令替代演示参数提取
- **服务管理**：用 `systemctl status` 替代 `start/stop`
- **安装软件**：用 `apt-get update` 替代完整安装

### 4. 常见问题处理

**Q: API 调用失败**
- 检查 `.env` 中的 API Key
- 确认网络连接正常

**Q: 命令执行权限不足**
- 使用 `sudo` 运行服务
- 或在演示时选择无需 root 的命令

**Q: 响应慢**
- LLM 调用需要 3-5 秒
- 利用这个时间解释正在发生什么
